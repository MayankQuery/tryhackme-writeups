# Notes – Dev Diaries

---

## 🔹 What is OSINT?

OSINT (Open Source Intelligence) involves collecting information from publicly available sources.

Examples:
- Websites
- GitHub
- Social media
- Search engines

---

## 🔹 Key Concepts in This Room

### Subdomain Enumeration
- Developers often leave staging or testing environments exposed
- Example:
  - uat-testing.marvenly.com

---

### GitHub OSINT

- Developers may expose:
  - Emails
  - Source code
  - Secrets
  - Commit messages

---

### Commit History Analysis

- Even deleted data can be recovered
- Git keeps history unless cleaned

---

## 🔹 Information Discovered

- Developer username
- Email address
- Commit history message
- Hidden flag

---

## 🔐 Security Risk

Exposed development traces can lead to:
- Data leakage
- Credential exposure
- Infrastructure compromise

---

## 🧠 Key Takeaway

> Publicly available development artifacts can reveal critical sensitive information if not properly secured.
